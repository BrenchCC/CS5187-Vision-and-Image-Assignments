# 项目作业：现实环境中的特征检测与匹配（占总成绩20%的8%）

## 1. 概述

在本项目中，您将从零开始构建一个完整的特征检测与匹配流水线，利用前几周课堂上涵盖的概念（滤波、Harris角点、特征不变性、描述符和RANSAC）。

虽然特征匹配在简单、受控的图像上可以完美工作，但现实世界的视觉应用面临着许多挑战。您的任务是实现该算法，并通过在10种不同压力测试场景中的5种上捕获和匹配图像对，来评估其鲁棒性。

请在报告中注明您的队友（如有）。

## 2. 实现要求（流水线）

您必须编写代码（Python/NumPy）来实现以下阶段：

1. **特征检测**：实现Harris角点检测器。您需要计算图像梯度、计算结构张量矩阵、应用高斯窗口，并找到Harris响应函数的局部最大值。

2. **特征描述**：在每个检测到的关键点周围提取局部描述符。您可以实现简化版的SIFT描述符或多尺度定向块（MOPS）。您的描述符应尝试考虑旋转和尺度不变性。

3. **特征匹配**：在两幅图像之间匹配描述符。实现平方差和（SSD）来找到最近邻，并应用Lowe比率测试来丢弃模糊的匹配。

4. **异常值剔除**：实现RANSAC算法来鲁棒地估计两幅图像之间的几何变换（例如仿射或单应性），并过滤掉不正确的匹配（异常值）。

## 3. 10种场景（数据集收集与评估）选择5种

您可以使用相机/智能手机捕获10种不同图像对中的5种（或从互联网获取）。每对应表示完全相同的场景或物体，但受到以下10种变换/挑战之一的影响。

在每对上运行您的流水线，并输出一幅在匹配特征之间绘制连线的图像。

1. **基准（平移）**：简单的水平或垂直相机移位。（预期结果：高精度）。

2. **尺度/缩放**：同一物体的一张广角镜头和一张特写镜头。

3. **平面内旋转**：将相机绕其轴旋转90或180度。

4. **透视/视角变化**：将相机向侧面移动约30-45度，引入平面外透视畸变。

5. **光照变化**：在完全不同的光照条件下拍摄的同一场景（例如，白天与夜晚，或开灯与关灯）。

6. **对焦/模糊**：一张完全清晰的图像，和一张失焦或有相机抖动运动模糊的图像。

7. **重复模式**：具有重复纹理的主体的图像对（例如砖墙、具有相同窗户的摩天大楼或棋盘）。这将测试您的比率测试是否正常工作。

8. **低纹理环境**：具有极少独特角点的场景（例如，只有一张海报的空白墙壁，或光滑的现代办公桌）。

9. **严重杂乱**：在干净环境中的物体与放在高度杂乱、混乱背景中的同一物体进行匹配。

10. **部分遮挡**：目标物体在第一张图像中完全可见，但在第二张图像中被另一个物体覆盖40-50%的图像对。

## 4. 交付物

### A. 代码提交
- 提交所有Python脚本/Jupyter Notebook。
- 您的代码必须干净、有良好注释且模块化（例如，`get_harris_corners()`、`get_descriptors()`、`match_features()`、`ransac()`等独立函数）。
- 包含一个`main.py`或等效文件，评分者可以轻松运行以在给定的图像对上重现您的结果。

### B. 结果图像
- 提交一个包含您20张原始输入图像的ZIP文件夹（10种场景每种2张）。
- 提交一个包含10张输出图像的文件夹。每张输出图像应将输入对并排放置，并绘制连接内点关键点的线条（RANSAC后）。
- 清晰标记所有文件（例如，`scenario_3_rotation_inputA.jpg`、`scenario_3_rotation_output.jpg`）。

### C. 项目报告（PDF）
您的报告（约4-6页）必须包括：

1. **流水线细节**：简要解释您的实现选择（例如，窗口大小、角点检测阈值、RANSAC迭代次数）。

2. **场景分析**：对于10种场景中的每一种，包括输出图像和一段简短的讨论性能的段落。
   - 算法是成功还是失败？
   - 如果失败或匹配很少，流水线的哪个部分出了问题（例如，Harris未能找到角点，还是描述符未能匹配）？

3. **局限性与改进**：基于您的结果，您的流水线的根本局限性是什么？您会添加哪些高级技术（例如深度学习特征、完整SIFT）来修复您表现最差的场景？

## 5. 评分标准（共100分）

- **特征检测与描述（25分）**：正确实现Harris角点和不变块提取。
- **匹配与RANSAC（25分）**：正确实现Lowe比率测试和鲁棒的单应性/仿射估计。
- **数据集收集（10分）**：捕获10组有创意、独特且合适的图像对。
- **结果图像（15分）**：匹配的视觉质量；内点与外点的清晰可视化。
- **报告与分析（25分）**：评估的深度。高分将给予那些彻底诊断为什么某些场景（如重复模式或严重模糊）导致算法失败的学生，而不仅仅是说明它失败了。
