Project Assignment: Feature Detection & Matching in the Wild （8% out of 20%）
1. Overview
In this project, you will build a complete feature detection and matching pipeline from
scratch, utilizing the concepts covered in the first few weeks of class (Filtering, Harris
Corners, Feature Invariance, Descriptors, and RANSAC).
While feature matching works perfectly on simple, controlled images, real-world vision
applications face numerous challenges. Your task is to implement the algorithm and
evaluate its robustness by capturing and matching image pairs across 5 out of 10
different stress-test scenarios.
Please note your teammate (if have) on the reports.
2. Implementation Requirements (The Pipeline)
You must write code (Python/NumPy) to implement the following stages:
1. Feature Detection: Implement the Harris Corner Detector. You will need to
compute image gradients, calculate the structure tensor matrix, apply a
Gaussian window, and find local maxima of the Harris response function.
2. Feature Description: Extract a local descriptor around each detected keypoint.
You can implement a simplified SIFT descriptor or Multi-Scale Oriented Patches
(MOPS). Your descriptor should attempt to account for rotation and scale
invariance.
3. Feature Matching: Match descriptors between two images. Implement the Sum
of Squared Differences (SSD) to find nearest neighbors, and apply Lowe’s
Ratio Test to drop ambiguous matches.
4. Outlier Rejection: Implement the RANSAC algorithm to robustly estimate a
geometric transformation (e.g., affine or homography) between the two images
and filter out incorrect matches (outliers).
3. The 10 Scenarios (Dataset Collection & Evaluation) select 5.
You can use your camera/smartphone to capture 5 out of 10 distinct pairs of images (or
from internet). Each pair should represent the exact same scene or object, but altered
by one of the following 10 transformations/challenges.
Run your pipeline on each pair and output an image drawing lines between matched
features.
1. Baseline (Translation): A simple horizontal or vertical camera shift. (Expected

result: High accuracy).
2. Scale / Zoom: One wide-angle shot and one zoomed-in shot of the same
object.
3. In-Plane Rotation: Rotate the camera 90 or 180 degrees around its axis.
4. Perspective / Viewpoint Change: Move the camera to the side by about 30–45
degrees to introduce an out-of-plane perspective distortion.
5. Illumination Variation: The same scene captured under drastically different
lighting (e.g., daylight vs. nighttime, or lights on vs. off).
6. Focus / Blur: One perfectly sharp image, and one image that is out of focus or
suffers from camera-shake motion blur.
7. Repetitive Patterns: An image pair of a subject with repeating textures (e.g., a
brick wall, a skyscraper with identical windows, or a checkerboard). This tests if
your ratio test works correctly.
8. Low-Texture Environment: A scene with very few distinct corners (e.g., a blank
wall with a single poster, or a smooth modern desk).
9. Heavy Clutter: An object in a clean environment matched against the same
object placed in a highly cluttered, messy background.
10. Partial Occlusion: An image pair where the target object is fully visible in the
first image, but 40-50% covered by another object in the second image.
